# REOP CRM — Claude Code Reference

## Product: SphereSync
One AI Coach agent driving a 3-tab hub. The hub lives at `/spheresync-tasks` (route) → `src/pages/SphereSyncTasks.tsx`. Tabs: **SphereSync | Pipeline | Database**. The admin Dashboard (`/`) is a separate admin-only surface.

New features go into a tab lens or a Coach action — not a new page.

## Stack
- **Build:** Vite + React 18 + TypeScript, **bun** as package manager. Path alias `@/` → `src/`
- **UI:** shadcn/ui + Radix primitives + Tailwind CSS + Lucide icons. Toasts via `sonner`. Forms: `react-hook-form` + `zod`
- **Data:** Supabase (Postgres + Auth + RLS + Edge Functions). TanStack Query v5 for server state (`staleTime: 60_000` globally)
- **Routing:** `react-router-dom` v7. New routes must use `React.lazy` + `<Suspense>`
- **Tests:** Vitest + React Testing Library (`bun run test:run`), Playwright for e2e

## AI Rule
**Default to xAI Grok** (OpenAI-compatible API, `grok-3-mini` or `grok-4-1-fast-reasoning`) in all edge functions.  
**Reserve `claude-opus-4-7`** for weekly agent intelligence synthesis only.  
Grok is 5–8× cheaper. Always persist `ai_model` field for auditability.

- `sphere_contacts` — contact universe with priority scores
- `pipeline_opportunities` — pipeline deals
- `ai_coach_tasks` — tasks generated by Coach agent

## Edge Functions (`supabase/functions/`)
81 functions total. Key ones:
- `ai-coach-agent` — main Coach agent (Grok)
- `spheresync-generate-tasks` — task generation
- `pipeline-score-opportunities` — opportunity scoring
- `generate-agent-intelligence` — admin intelligence (Claude, weekly)

## Conventions
- **Components** live in `src/components/<domain>/` — no new top-level domains unless it's a new product area
- **Hooks** in `src/hooks/use<Feature>.ts` — TanStack Query, not raw `useEffect`+`useState`
- **Migrations** in `supabase/migrations/YYYYMMDDHHMMSS_<uuid>.sql` — always idempotent, always include RLS
- **Never hand-edit** `src/integrations/supabase/types.ts`
- **Never bypass RLS** from the client (no service-role key in browser)
- PRs: worktree per feature → PR into `main`, small + focused

## Routing
- `/` → Admin Dashboard (admin only; agents redirected to `/spheresync-tasks`)
- `/spheresync-tasks` → SphereSync hub (3-tab, agent default)
- `/database`, `/events`, `/newsletter`, `/coaching`, `/transactions`, `/social-scheduler`, `/settings`, `/support`
- `/admin/*` → Admin-only management pages

## Performance Defaults (non-negotiable)
1. TanStack Query for all server state — `queryKey` as tree, not string blob
2. `React.lazy` + `<Suspense>` for all new routes
3. Virtualize lists > ~100 rows with `@tanstack/react-virtual`
4. Optimistic updates for all high-frequency clicks (pipeline drag, task check-off)
5. Server-side pagination with `.range()` — never unbounded `SELECT *`
